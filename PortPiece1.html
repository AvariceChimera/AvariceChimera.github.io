• made in C++ using a custom built framework that makes use of SDL/OpenGL libraries.

  • used a randomizing seed to make a grid and fill it with tiles for a level

  • was meant to have a 10 minute timer that would have casued a game over if it ran out but due to time constraints and tunnle vision on bug fixing it couldn't be implemented properly

This was my first big solo project that wasn't a tutorial/making galaga I used SDL/OpenGL to build it from the ground up.
Sadly this project didn't make it to a playable state (time constraints) I'm still proud of it nonetheless.
When designing things i kinda cheated by making the base floor tile a checker patern it gave me a clear count of pixels.
As for the stuf in the tiles i had to force myself to accept imperfections because i kept getting caught on making everything centered.
After that was picking the types of gems, figuring out how to make them look, and how to score each one on it's own.
It was going to be a combitation puzzle/speed game. You would move about a meuseum exhibit trying to steal jewels and
make an escape via a drainage hatch (the grey circles with a big "E" on them). You could grab one or all five but
you couldn't leave empty-handed more jewels = more points and all that jazz. Development wasn't that bad, BUGFIXING HOWEVER
now that was rough honestly it was what caused me to miss the deadline. I was trying to fix a bug with movement and
the level spawning properly randomized (funnily enough that was the easiest part) and I was so focused on the bug hunt I
forgot the rest of the game. And by the time I wanted to get back to it we had to work on the next project so I
didn't have the time or ability to finish it. However I did make the assests myself. (though I did have them touched
up by my S.O. when I was feeling burnt out) My art isn't top notch but I like to think I have a good eye for it.

I really learned a lot through this project and I feel it was the first big step I took towards finding my niche.
I could easily finish it on my own time but I personally won't. As if I were to pick this up again I want to put it in
a proper engine like unity or unreal for sake of the expanded resources that are at my disposal in that case.
I don't plan on looking back largely in part to what I've learned I want to start over and try new ideas with this project.
